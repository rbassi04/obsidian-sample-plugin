# State management
When you do `let n = "asnwer"; n = "Note"` you lose the old value. To support features like undoing and redoing changes, Obsidian keeps a history of changes.

```js
const changes: ChangeSpec[] = [];

changes.push({ from: 0, insert: 'Heading' });
changes.push({ from: 0, insert: '# ' });
changes.push({ from: 0, insert: '#' });
```

## Transactions:
For editor extensions, a group of state changes that happen together is called a `transaction`.

If you select text and then double quote it, e.g. `Heading` -> `"Heading"`. 

How you'd add, or dispatch, a transaction to the editor view:
```js
view.dispatch({
  changes: [
    { from: selectionStart, insert: `"` },
    { from: selectionEnd, insert: `"` }
  ]
});
```


# State fields
State fields is an editor extension which lets you manage customer editor state.

## Defining state effects:
State effect is the state change you'd like to make. Like methods on a class.

```js
// input type number
const addEffect = StateEffect.define<number>();
// input type number
const subtractEffect = StateEffect.define<number>();
const resetEffect = StateEffect.define();
```

## Defining a state field
Doesn't store state, it manages it. Takes the current state, applied any state effect, returns the new state.

It has the logic.

A transaction can contain multiple effects, for example two additions, the state field needs to apply them all one after another.

```js
export const calculatorField = StateField.define<number>({
  create(state: EditorState): number {
    return 0;
  },
  update(oldState: number, transaction: Transaction): number {
    let newState = oldState;

    for (let effect of transaction.effects) {
      if (effect.is(addEffect)) {
        newState += effect.value;
      } else if (effect.is(subtractEffect)) {
        newState -= effect.value;
      } else if (effect.is(resetEffect)) {
        newState = 0;
      }
    }

    return newState;
  },
});
```

1. `create` returns the value the calculator starts with.
2. `update` contains the logic for applying the effects.
3. `effect.is()` lets you check the type of the effect before you apply it.

## Dispatching state effects
To apply a state effect to a state field, you need to dispatch it to the editor view as part of a transaction.

```js
view.dispatch({
  effects: [addEffect.of(num)],
});
```

You can even define a set of helper functions that provide a more familiar API:

```js
export function add(view: EditorView, num: number) {
  view.dispatch({
    effects: [addEffect.of(num)],
  });
}

export function subtract(view: EditorView, num: number) {
  view.dispatch({
    effects: [subtractEffect.of(num)],
  });
}

export function reset(view: EditorView) {
  view.dispatch({
    effects: [resetEffect.of(null)],
  });
}
```
