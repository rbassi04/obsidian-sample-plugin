# Start:

```shell
cd path/to/vault
mkdir .obsidian/plugins
cd .obsidian/plugins

git clone https://github.com/obsidianmd/obsidian-sample-plugin.git
```


**Build the plugin**
```shell
cd obsidian-sample-plugin
npm install
npm run dev
```

**Enable the plugin**
Turn on in Obsidian's Settings

**Update plugin manifest**
To change plugin info like name and description

1. In `manifest.json` change id (unique id) to your own
2. In `manifest.json` change name to the visible name
3. Rename the plugin's folder name to the id
4. Restart plugin

**Anaotmy of Plugin**
* `onload()` runs whenever the user starts using the plugin in Obsidian. This is where you'll configure most of the plugin's capabilities.

* `onunload()` runs when the plugin is disabled. Any resources that your plugin is using must be released here to avoid affecting the performance of Obsidian after your plugin has been disabled.
